# HackerTrip 小程序 · Neo-Brutalist 设计基础规范（Design Tokens）

> **主题锁定 2026-06-10**：新野兽派 Neo-Brutalist Blue。
> **唯一风格真源**：`design/prd-design-v2/effect-mockups/v8-theme-explore/theme-neobrutalist-blue.png`（所有 token 从该效果图取色/量取）。
> **作废**：旧版 `hackertrip-design-system.md` 的全部配色（暖白 `#FAF9F5` + 紫色 `#4D4DE9` / `#EFEDFF` / mint `#BDF6CC` 等）一律不再使用。旧版的页面结构/内容仍可作为信息参考。
>
> **本文件职责边界**：只定义「基础 token 层」——颜色 / 边框 / 阴影 / 字体 / 间距 / 圆角 / 状态原则 / 动效曲线。
> **不定义具体组件规格**（Button / Card / Chip / StatusPanel 等的精确尺寸与构造由组件层 Agent 负责，避免重复与冲突）。
>

## 1. Color Tokens

> 60-30-10 法则：**60% = 背景米白 + 白卡面**（载体），**30% = 纯黑边框/黑字/黑底标签**（结构骨架，野兽派的"重"由黑边承担），**10% = 电光蓝主色 + 柠檬黄强调**（行动与点睛）。蓝是唯一主操作色，黄是稀缺点睛色（仅角标/选中/极小面积），绝不大面积铺黄。

### 1.1 背景与面 Surface（占比 ~60%）

| Token | Hex | 用途 | 占比建议 |
| --- | --- | --- | --- |
| `color.bg.canvas` | `#F4F1E8` | 页面底（亮米白 off-white，野兽派暖底，与纯白卡面拉开层次） | 背景大面积 |
| `color.bg.surface` | `#FFFFFF` | 卡片、命令栏、输入框、面板（纯白，靠黑边+硬投影浮起） | 主要内容载体 |
| `color.bg.sunken` | `#ECE8DB` | 进度条槽、输入框内凹、次级底（比 canvas 略深的米白） | 局部 |
| `color.bg.blueFill` | `#0C51ED` | 状态面板/身份卡的整块蓝底（蓝当背景的大色块场景） | ≤10%，归入主色配额 |

> 说明：效果图渲染把底色压到了近白 `#F8F8F8`；按锁定主题"亮米白 off-white"基调，正式取 `#F4F1E8`（米白带暖、非纯白、非纯灰），保留野兽派纸感。若团队倾向更冷更白，可退回 `#F7F6F2`，但不要用纯 `#FFFFFF` 当页面底（会和卡面糊在一起，黑边对比也变脏）。

### 1.2 主色 Primary（电光蓝，占比 ~主色配额内）

| Token | Hex | 用途 |
| --- | --- | --- |
| `color.brand.primary` | `#0C51ED` | 主操作 / 主按钮底 / 选中态 / 进度已完成段 / 链接 / 蓝色大色块 / tabbar 选中（取自效果图最高频蓝） |
| `color.brand.primaryPressed` | `#0A41BE` | 按压态（primary 压暗 ~22%，配合硬投影位移使用） |
| `color.brand.primaryHover` | `#2E6AF0` | hover / 次强调（小程序少用 hover，主要给设计稿/H5 预览） |
| `color.brand.onPrimary` | `#FFFFFF` | 蓝底之上的文字/图标（纯白） |

### 1.3 强调 Accent（柠檬黄，占比 ~点睛 ≤5%）

| Token | Hex | 用途 |
| --- | --- | --- |
| `color.accent.lemon` | `#FBD902` | 柠檬黄强调：选中 chip 底（如「全部」）、角标、"NEW/HOT"小标、稀缺点睛色块（取自效果图实测黄） |
| `color.accent.lemonPressed` | `#E0C200` | 黄底元素按压态 |
| `color.accent.onLemon` | `#000000` | 黄底之上一律用纯黑文字（黄底配白字对比不足，禁用） |

> 黄是"点睛"不是"主色"：单屏黄色面积建议 < 5%。大按钮、大色块用蓝，黄只做小面积高亮与角标。

### 1.4 语义色 Semantic

| Token | Hex | 用途 | 占比 |
| --- | --- | --- | --- |
| `color.success` | `#13B36B` | 成功 / 已完成 / 高匹配分文字（野兽派饱和绿，配黑边） | 点睛 |
| `color.success.fill` | `#3DDC84` | 成功填充块（如已完成进度段的可选高饱和变体） | 点睛 |
| `color.urgent` | `#FF4D2E` | 紧急 / 截止 <24h 倒计时 / 报名截止（高饱和橙红，野兽派警示色） | 点睛 ≤3% |
| `color.urgent.fill` | `#FF4D2E` | 紧急角标/倒计时底（黑边 + 白字或黑字按对比择优） | 点睛 |
| `color.heat` | `#FF7A00` | 热度 🔥 数值（橙，区别于 urgent 红，避免误读为紧急） | 点睛 |

> 语义色全部走"黑边 + 高饱和填充"的野兽派表达，绝不用淡色/柔和色。绿/红/橙互不混用：红=时间紧急，橙=热度人气，绿=完成/高分。

### 1.5 文字 Text（三级）

| Token | Hex | 用途 |
| --- | --- | --- |
| `color.text.primary` | `#000000` | 主标题 / 正文强调 / 卡内核心信息（纯黑，野兽派标志性重黑字） |
| `color.text.secondary` | `#3D3D3D` | 正文 / 描述 / 元信息（深炭灰，仍偏重） |
| `color.text.tertiary` | `#7A7A75` | 次要元信息 / 占位 placeholder / 禁用文字（暖灰，配米白底） |
| `color.text.onDark` | `#FFFFFF` | 蓝底/黑底之上的文字 |
| `color.text.onLemon` | `#000000` | 黄底之上的文字（强制纯黑） |

> 野兽派不走"灰阶层次"，主信息一律纯黑，靠**字重和字号**拉层次而非靠浅灰。tertiary 仅用于真正的弱信息（时间戳、placeholder）。

### 1.6 边框与分隔 Border / Line

| Token | Hex | 用途 |
| --- | --- | --- |
| `color.border.ink` | `#000000` | 标准黑边（所有元素 2px 纯黑实线，野兽派核心） |
| `color.border.inkSoft` | `#000000 @ 1px` | 卡内次级分隔（仍纯黑，仅减宽到 1px，见 §2） |
| `color.shadow.ink` | `#000000` | 硬投影颜色（纯黑，无 alpha、无模糊，见 §3） |
| `color.line.divider` | `#000000` | 列表行/区块分隔线（纯黑 1px，必要时 `#1A1A1A`） |

> 野兽派**没有"浅灰边框"**。所有边/线/分隔/投影统一用纯黑，区别只在"宽度"和"是否硬投影"，不在颜色深浅。禁止出现 `rgba(0,0,0,0.08)` 这类淡边（那是上一版暖白·紫的做法，已作废）。

## 2. Border 边框规范

野兽派的"重"全靠黑边，宽度是体系的关键变量。

| 层级 | 宽度 | 颜色 | 适用 |
| --- | --- | --- | --- |
| `border.heavy` | **2.5px** | `#000` | 首屏主元素：状态面板、精选大卡、主按钮、搜索栏、tabbar 容器（最重，承载主视觉骨架） |
| `border.base` | **2px** | `#000` | 标准元素：普通卡片、输入框、chip、列表项、角标（默认值，绝大多数元素用这个） |
| `border.thin` | **1px** | `#000` | 卡内次级分隔、紧凑标签、小图标描边（密集信息区减重，避免黑边糊成块） |

规则：
- **每个独立可视元素都要有黑边**（卡片、按钮、chip、输入框、角标、头像框、进度条槽、tabbar），这是野兽派的硬性特征。
- 边框 `border-style: solid`，**永远纯黑、永不圆滑渐变、永不半透明**。
- 嵌套时外层用 `heavy/base`、内层用 `thin`，避免两层 2px 黑边贴在一起显脏。
- 描边与圆角配合：黑边走在圆角外沿（`box-sizing: border-box`），圆角值见 §5。

## 3. Shadow 硬投影规范（Hard Shadow）

野兽派标志性的**硬边偏移黑投影**：纯黑、零模糊、固定方向偏移，制造"实体卡片悬在纸上"的剪贴感。

```
统一方向：向右下偏移（光从左上来）
统一格式：box-shadow: {offset}px {offset}px 0 0 #000;   // 第三参 blur 永远为 0
```

| Token | 值 | 用途 |
| --- | --- | --- |
| `shadow.sm` | `2px 2px 0 0 #000` | 小元素：chip、角标、小按钮、标签 |
| `shadow.md` | `4px 4px 0 0 #000` | 标准卡片、输入框、列表卡、次级按钮（默认硬投影） |
| `shadow.lg` | `6px 6px 0 0 #000` | 首屏主元素：状态面板、精选大卡、主按钮、命令栏、tabbar |
| `shadow.xl` | `8px 8px 0 0 #000` | 浮层 / 弹窗 / 抽屉 / 分享身份卡（最高层级，最强剪贴感） |

按压态投影变化（核心交互语言）：
- 元素被按下时，**投影收缩 + 元素位移到投影位置**，制造"被按实"的物理感：
  ```
  默认:   transform: translate(0,0);        box-shadow: 4px 4px 0 0 #000;
  按压:   transform: translate(2px,2px);    box-shadow: 2px 2px 0 0 #000;
  // 即：元素向右下移动 = 投影偏移量的一半，投影同步减半
  ```
- 主元素（lg `6px`）按压 → `translate(3px,3px)` + `shadow 3px 3px`。
- 禁用态：去掉硬投影，只保留 2px 黑边 + 降低不透明度（见 §6）。

规则：
- 投影颜色**恒为纯黑 `#000`**，无 alpha、blur 恒为 0、扩散恒为 0。
- 方向**全局统一右下**（`+x +y`），禁止某些元素左上、某些右下，否则光源混乱。
- 硬投影**只给"浮起的实体"**（卡片、按钮、面板、角标）；纯文字、分隔线、背景块不加投影。
- 投影不参与 elevation 的"模糊扩散"逻辑——野兽派没有 Material 式柔影，层级只由 **偏移量大小** 区分（见 §6）。

## 4. Typography 字体规范

### 4.1 字体栈（中英文回落）

```
中文：    "PingFang SC", "Heiti SC", sans-serif        // 小程序中文一律 PingFang SC
拉丁/数字：重 grotesk 风格几何无衬线；小程序回落系统栈
完整栈：  -apple-system, "SF Pro Display", "PingFang SC", "Helvetica Neue", Arial, sans-serif
```

> 设计稿可用 Space Grotesk / Archivo / Inter Tight 等重 grotesk 表现拉丁与数字（倒计时「12」、匹配度「92%」等大数字尤其需要重 grotesk 的硬朗感）；小程序端因不能内嵌大字库，回落系统 `-apple-system`，**用字重 Heavy/Bold 补足野兽派的"重"**。

### 4.2 字阶（px@393）

| 角色 | 字号 | 字重 | 行高 | 用途 |
| --- | --- | --- | --- | --- |
| `font.title` | **30px** | **Heavy / 900** | 1.15 | 页面主标题「发现黑客松」、超大数字（倒计时） |
| `font.subtitle` | **22px** | **Bold / 800** | 1.2 | 卡片大标题、状态面板赛事名、区块标题「精选推荐」 |
| `font.heading` | **17px** | **Bold / 700** | 1.3 | 卡片标题、列表项主文、按钮文字 |
| `font.body` | **15px** | Semibold / 600 | 1.5 | 正文、描述、placeholder（野兽派正文也偏重，最低 600） |
| `font.secondary` | **13px** | Medium / 500 | 1.4 | 元信息（日期/城市/形式）、次要说明 |
| `font.small` | **11px** | Bold / 700 | 1.25 | 角标、tag、chip 文字、tabbar 文字（小字反而加粗，保证野兽派的"硬"） |

规则：
- **整体字重比常规设计高一档**：常规系统 Regular 的位置，这里用 Medium/Semibold；常规 Medium 的位置用 Bold。野兽派靠"粗"立威。
- 标题类（title/subtitle/heading）一律 **Bold 及以上**，大标题冲 Heavy/900。
- 中文用 PingFang SC 的 `font-weight` 在小程序里有效档位有限（常规/中粗/粗），title 级可叠加描边或更大字号补偿"重"。
- 行高克制（1.15–1.5），野兽派紧凑、密实，不要松垮的大行距。
- 字色默认 `text.primary #000`；蓝底/黑底用 `onDark #FFF`；黄底用 `onLemon #000`。

## 5. Spacing 间距规范

刻度（px@393，×2 ≈ rpx 落地）：

```
4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 56 · 72
```

| 场景 | 值 | 说明 |
| --- | --- | --- |
| 页边距（左右安全边） | **16px** | 比上一版 20px 略收，给大黑边+硬投影留呼吸但保持密实 |
| 卡内边距 padding | **16px**（紧凑卡 12px，大面板 20px） | 黑边内侧到内容 |
| 卡片之间间距 | **12px** | 列表卡纵向间距；注意要 ≥ 硬投影偏移量(4–6px) + 4px，避免投影压到下一张 |
| 区块之间间距 | **24–32px** | 「正在关注」「精选推荐」「更多黑客松」等区块标题上方 |
| 元素内 chip/tag 间距 | **8px** | 同行多个 chip/tag |
| 图标与文字间距 | **6–8px** | icon + label |
| 网格卡横向间距 | **12px** | 精选三列卡之间 |

规则：
- **硬投影占空间**：相邻元素间距必须 > 投影偏移量，否则下一张卡的黑边会被上一张的硬投影盖住。`shadow.md(4px)` → 间距 ≥ 12px；`shadow.lg(6px)` → 间距 ≥ 16px。
- 容器右/下侧要预留投影空间（外层 padding 或 margin 加 `投影偏移量`），避免投影被屏幕边缘/父容器裁切。
- 间距节奏密实、规整，呼应野兽派"块状、对齐、网格"的气质，不要随意的奇数间距。

## 6. Radius 圆角规范

野兽派用 **chunky 圆角**——圆但不软，配合大黑边形成"厚墩墩"的实体感。

| Token | 值 | 用途 |
| --- | --- | --- |
| `radius.xs` | **6px** | 小角标、紧凑 tag |
| `radius.sm` | **10px** | chip、小按钮 |
| `radius.md` | **12px** | 输入框、列表行、普通卡片、图片块 |
| `radius.lg` | **16px** | 主卡片、精选大卡、状态面板内块 |
| `radius.xl` | **20px** | 状态面板 / 命令栏 / 身份卡 / 抽屉等大容器 |
| `radius.pill` | **999px** | 全圆 chip、圆形按钮（搜索/发送 FAB）、头像 |

规则：
- 圆角与黑边、硬投影**三件套绑定**：圆角矩形 + 2px 黑边沿圆角走 + 右下硬投影同圆角。三者圆角半径一致，投影边缘跟随圆角（`box-shadow` 默认跟随 border-radius）。
- 不要过大圆角（>24px 会失去野兽派的"块感"，变可爱风）；也不要纯直角（0px 太生硬冷峻，偏工业 brutalist 而非 neo-brutalist 的活泼）。**chunky 的甜区是 10–20px**。
- 嵌套时内层圆角 ≤ 外层（内层 12 / 外层 16），保持同心。

## 7. Elevation / States 层级与状态

### 7.1 Elevation（硬投影分层）

野兽派的 elevation **不靠模糊扩散，只靠硬投影偏移量大小**区分层级：

| 层级 | 投影 | 元素 |
| --- | --- | --- |
| L0 平面 | 无投影，仅在 canvas 上 | 背景块、分隔线、纯文字 |
| L1 基础卡 | `shadow.md 4px` | 列表卡、输入框、chip |
| L2 主元素 | `shadow.lg 6px` | 状态面板、精选大卡、主按钮、命令栏、tabbar |
| L3 浮层 | `shadow.xl 8px` + 背后半透明黑遮罩 | 弹窗、抽屉、分享身份卡、底部动作面板 |

> 越"高"的层投影偏移越大，黑边可同步升到 2.5px。遮罩用 `rgba(0,0,0,0.45)`（纯黑半透明，不用模糊背板/毛玻璃——那不是野兽派）。

### 7.2 各态视觉处理原则（野兽派表达）

| 状态 | 处理原则 |
| --- | --- |
| **Loading 骨架** | 用**纯黑边 + 米白/浅灰实底块**的骨架（对齐真实卡片布局，带黑边和硬投影占位），骨架块做 `#ECE8DB ↔ #F4F1E8` 的硬切换闪烁（**不要柔和的渐变 shimmer**，野兽派要"硬"的脉冲）。禁用纯 spinner。 |
| **Empty 空态** | 居中：一个**带黑边的方形/圆形图标块**（蓝底或黄底）+ 一句重黑字文案 + 一个主按钮（蓝底白字 + 黑边 + 硬投影）。空态也要有"实体感"，不要灰扑扑的轻插画。 |
| **Error 错误** | **内联重试**：在内容位置放一个带黑边的卡，红色 `urgent` 角标 + 重黑字错误说明 + 「重试」按钮（黑边+硬投影）。保留顶栏/命令栏可用，不整屏报错。 |
| **Offline 离线** | 顶部贴一条**满宽黑边横幅**：黄底 `accent.lemon` + 黑字「网络不可用，下拉重试」+ 黑边 + 硬投影（黄=醒目警示，黑字保证可读）。有缓存则下方照常展示缓存内容。 |
| **Unauthed 未登录** | 游客只读，内容正常展示；**仅在写操作处**（收藏/报名/生成身份卡）触发登录——弹一个带黑边硬投影的小卡，一行理由 + 蓝色主按钮，不在入口就拦。 |
| **Disabled 禁用** | 去掉硬投影（变平），保留 2px 黑边，整体 `opacity: 0.4`，禁止按压位移动画。 |
| **Pressed 按压** | 见 §3：元素向右下 `translate` + 投影同步收缩（核心交互反馈，所有可点元素统一）。 |
| **Selected 选中** | chip/tab 选中 → 底色切到 `accent.lemon` 黄 或 `brand.primary` 蓝（按层级），黑边加重或加内描边，文字按底色取 `#000`/`#FFF`。 |

## 8. Motion 动效规范

野兽派动效**克制、短、硬、有物理感**——服务于"被按实"的实体反馈，不做花哨缓动。

| 交互 | 动效 | 时长 | 缓动 |
| --- | --- | --- | --- |
| 按压（点击反馈） | `translate(0,0)→(2px,2px)`（主元素 3px）+ 硬投影同步收缩 | **80–120ms** | `cubic-bezier(0.2, 0, 0, 1)`（快出，干脆） |
| 抬起（松手回弹） | 反向回到默认位 + 投影恢复 | **120ms** | `cubic-bezier(0.3, 0, 0.2, 1)`（带极轻回弹感但不 overshoot 过头） |
| hover（仅 H5/设计稿） | 元素上抬 `translate(-1px,-1px)` + 投影 +1px（"浮更高"） | 100ms | `ease-out` |
| chip/tab 选中切换 | 底色 + 字色 硬切，无渐变过渡 | **0ms（瞬切）** 或 ≤80ms | linear |
| 弹层/抽屉进出 | 从边缘 `translate` 滑入 + 遮罩 0→0.45 | 200ms | `cubic-bezier(0.2, 0, 0, 1)` |
| 骨架闪烁 | `#ECE8DB ↔ #F4F1E8` 两态硬切 | 周期 800–1000ms | `steps(2)` 或硬切，**不用平滑渐变** |
| 进度条增长 | 已完成段宽度增长 | 300ms | `ease-out` |

规则：
- **首选 `translate` + 硬投影联动**表达交互，避免 Material 式的 ripple 水波、阴影柔变。
- **`scale` 谨慎用**：野兽派偏好"位移"而非"缩放"。如需缩放仅用 `scale(0.98)` 极轻量级，且只用于无投影的纯按钮，不与位移叠加。
- 缓动整体**偏快、偏线性、干脆收尾**，不要弹性 overshoot、不要慢悠悠 ease-in-out 长缓动。
- 选中态/筛选切换倾向**硬切（瞬变）**，强化野兽派"直接、不修饰"的气质。
- 所有动效 ≤ 200ms（弹层除外），保证"硬朗利落"的手感。

## 9. 关键约束（落地红线）

1. **黑边无处不在**：每个独立元素 2px（主元素 2.5px / 内层 1px）纯黑实边，禁用淡灰边/半透明边。
2. **硬投影零模糊**：`box-shadow` 第三参（blur）恒为 0，颜色恒纯黑，方向恒右下。
3. **黄不铺面**：柠檬黄仅点睛（选中/角标/小块），单屏 < 5%；蓝才是主操作色。
4. **不做暗色**：仅亮米白主题。
5. **不用毛玻璃/柔影/渐变光晕**：那是上一版（暖白·紫 / 玻璃拟态）的语言，与野兽派冲突，全部禁用。
6. **字偏重**：整体字重比常规高一档，标题 Bold+，靠粗细而非浅灰拉层次。
7. **间距让投影**：相邻元素间距 > 硬投影偏移量，容器预留投影空间防裁切。
8. **安全区**：顶部留微信胶囊安全区；底部 tabbar/命令栏留 home indicator；最小触达 88rpx(≈44px)。
9. 图片压缩转 WebP（参考项目历史 LCP 教训）。

## 11. 布局 · 留白 · 层次（从参考图反推，**最高优先级·防"夸张铺满"**）

> 真源参考图 = `effect-mockups/theme-neobrutalist-blue.png`（用户锁定并认可的发现页）。下列规则用来**纠正"每个模块都加黑框硬投影、从上到下铺满、没有主次"的塌方**。出任何页面、写任何组件，本章规则**压倒一切**——宁可少放内容、留白，也不要塞满。

### 11.1 一屏一主角（Focal Hero）
- **每个页面必须有且只有一个视觉焦点**，它在尺寸/色彩/权重上明显压过其余一切。其余模块一律"安静下来"做配角。
  - 首页焦点 = 「正在参赛/状态面板」那一张大卡（赛事名+倒计时+进度）。
  - 发现页焦点 = 「正在关注」面板 + 大标题区的品牌 H logo。
  - 匹配结果焦点 = 项目画像卡 或 #1 那条。
  - 详情页焦点 = 顶部赛事 hero 海报+标题。
  - 赛程焦点 = 距提交倒计时+进度那一块。
  - 身份卡焦点 = 那张大身份卡本身。
  - 同步焦点 = 6 位配对码输入区。
- 判断标准：眯眼看页面，**第一眼必须落在焦点上**。如果一眼分不清看哪，就是错的。

### 11.2 黑框/硬投影是"重音"，不是"标点"——只给关键卡
- ❌ 错（导致夸张）：**每个**元素（小数字块、每个快捷入口、每个 chip、每行）都套 2px 黑边+硬投影 → 满屏等重黑框打架。
- ✅ 对：**硬黑边+硬投影只给「主要内容卡」**——赛事海报卡、焦点状态面板、主 CTA 按钮、FAB。
- 次级元素（区块小标题、正文、统计数字、辅助说明、最近搜索词等）**用纯文字/极轻分隔**，不加黑框、不加硬投影。靠**字重+字号+留白**分层，不靠描边。
- 一屏内带"硬投影"的元素**建议 ≤ 4 个**。超过就开始糊。

### 11.3 留白与节奏（参考图的呼吸感）
- **区块间距 ≥ 28–32px**（比卡内间距大得多），让每个 section 之间有明显空气。
- 区块结构固定为：**小标题（borderless）→ 内容**，小标题与上一区块之间留足空白。
- **不要填满整屏高度**。底部留白、模块之间留白都是允许且鼓励的；宁可一屏只放 3–4 个区块，也不要硬塞 7–8 个。
- 页边距恒 16，但**重要焦点卡可适当更大留白衬托**。

### 11.4 用插画海报代替"文字方块"
- 凡是代表赛事/项目的卡，**视觉位用插画海报**（参考图里春潮/AdventureX/腾讯云那种带插画的贴纸海报），不要做成"纯文字塞进黑框"。视觉趣味来自海报，不来自更多黑框。

### 11.5 配色克制（焦点才上色）
- 蓝/黄是"重音色"：**主要给焦点和主 CTA**。次级区块以黑白+米白为主。
- 一屏内大面积蓝色块 ≤ 1–2 处；柠檬黄 < 5% 面积（角标/进度/单个 FAB）。避免每个模块都蓝一块黄一块。

### 11.6 出图/走查 checklist（每页必过）
- [ ] 眯眼第一眼能立刻找到唯一焦点？
- [ ] 带硬投影的元素 ≤ 4 个？次级元素是否已"去框"？
- [ ] 区块之间有明显留白、没有铺满整屏？
- [ ] 赛事卡用的是插画海报而非文字方块？
- [ ] 蓝/黄只在焦点和 CTA，没有满屏撞色？
- 任一条不过 = 重做，别将就。
