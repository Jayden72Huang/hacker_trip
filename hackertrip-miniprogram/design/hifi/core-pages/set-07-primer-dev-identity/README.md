## Why This Direction

Primer is built around developer workflows, repositories, issues, states, and identity. HackerTrip's roadmap includes GitHub binding, project cards, public profiles, skills libraries, and agent configs, so this direction aligns closely with the product architecture.

## Product Logic

- Primer blue handles links, primary action, and navigation.
- Green represents submitted/success states.
- Yellow is used for attention and deadlines.
- Project cards use repo-like metadata so identity and portfolio features feel natural.

## Fit For HackerTrip

Best for making HackerTrip feel like a developer identity and competition workspace, not just a hackathon feed. It supports GitHub, projects, skills, token usage, and portfolio modules especially well.

## Risk

It can feel too close to generic developer tooling if the visual assets are not distinctive enough.

## Recommendation

Recommended as a practical long-term system if GitHub/project identity becomes the main product wedge.
