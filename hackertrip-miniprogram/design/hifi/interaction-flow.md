# HackerTrip 小程序 · 交互链路与跳转逻辑（v2 · 已过 Codex 逻辑审查）

> **用途**：给 Figma 设计 Agent + 前端开发的唯一跳转真源。每条跳转有编号（J-xx），Figma 复刻时按编号在 frame 上标注热区 → 目标页；前端按编号实现路由。
> **导航定稿（2026-06-10，覆盖 PRD §12）**：底部 tabbar（发现 / 赛程 / 消息 / 我的），**无左抽屉**。PRD §12.1-12.2 的 drawer 方案作废，§12.3 跳转关系仍有效并已并入本文档。
> **配色/组件真源**：`neobrutalist-design-system.md` / `neobrutalist-components.md`。页面简报：`page-specs.md`。

## 2. 选手旅程（6 阶段 → 页面映射）

| 阶段 | 选手在做什么 | 主承接页 | 关键跳转 |
| --- | --- | --- | --- |
| ① 冷启动/逛 | 第一次打开，看看有什么比赛 | 发现（游客可读，不强制登录） | J-01/02/03 |
| ② 找赛事 | 搜索筛选，或让 AI 帮忙挑 | 发现 → 聊天 → 匹配结果 | J-04/05/10/11 |
| ③ 决策报名 | 看详情、问 AI 合不合适、**去官方报名**、加入赛程、设提醒 | 详情 | J-20~26 |
| ④ 参赛中 | 看阶段进度、做任务、收提醒 | 赛程 + 消息 | J-30~33, J-40/41 |
| ⑤ 提交冲刺 | 倒计时 <24h，检查提交就绪度、补齐提交材料 | 赛程（紧急态）+ 提交清单 | J-31/J-34/J-25 |
| ⑥ 赛后资产 | 更新身份卡、归档作品、分享拉队友 | 我的 → 身份卡/作品集 | J-50~56 |

闭环：⑥ 的分享落地页把新用户带回 ①（裂变），Skills 同步（07）把桌面 CLI 资产注入 ⑥。

## 4. 每页状态承接（防"点了没下文"）

| 页面 | 必须设计的分支 |
| --- | --- |
| 发现 | loading 骨架 / 空结果（CTA→J-04 AI 匹配）/ 正常 / 无关注（隐藏 J-05 条） |
| 赛程 | 有赛事 / 空态（J-33 双 CTA）/ **未登录**（"登录后查看你的参赛进度"，登录后回本页）/ 紧急红（J-34）/ 赛后（CTA→更新身份卡+归档作品） |

| 身份卡 | 空（J-53）/ 已生成 / 编辑中 / 未登录 |
| 同步 | 输入 / 等待 / 成功（J-61）/ 失败 / 过期 |

## 5. 给 Figma Agent 的标注规则（Step 4+ 执行约定）

1. 每个 frame 命名 = 页面编号（如 `05-event/normal`、`05-event/closed`）。
2. 每条跳转在源 frame 上加 Figma prototype reaction（`ON_CLICK → NAVIGATE` 到目标 frame）+ 一条注释文字 `J-xx → 目标`；半屏/弹层用 `OVERLAY`。
3. 焦点流页面（03/04/05/07）不放 tabbar，按 J-93 规则做返回（back 或首页锚点 fallback）；其余页 tabbar 高亮对应 tab。**06 身份卡归属「我的」域**：无论从发现 CTA 还是「我的」进入，tabbar 高亮「我的」。
4. 每个 frame 注明它承接的 intent（PRD §25.2），如 `05-event ⇐ event.detail / event.remind`。
