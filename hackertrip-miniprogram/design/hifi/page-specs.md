## P0-1 主页 · 发现黑客松 Discovery（首 tab，对标参考图）

- **目标**：落地即浏览真实赛事；AI 匹配作为显著入口，而非冷启动空屏。
- **核心模块**：顶部（定位城市「深圳 ▾」+ 微信胶囊）→ 品牌区（大标题「发现黑客松」+ 副标题 + AI 一句话 + 环带 H logo）→ 搜索框（蓝色搜索钮）→ 筛选 chips（全部/AI/硬件/出海/Web3/筛选）→「正在关注」状态条（有关注赛事时：封面缩略 + 倒计时 + 匹配度）→ 精选推荐走马灯（贴纸海报大卡 ≤3）→ 更多黑客松列表卡 →「生成我的身份卡」CTA banner → tabbar。
- **关键交互**：chip 筛选实时刷新；卡片进详情；收藏 ☆；搜索；身份卡 banner → 身份卡工坊；「正在关注」→ 赛程 tab。
- **状态**：loading 骨架 / 空结果（引导 AI 匹配）/ 正常列表 / 无关注赛事（隐藏「正在关注」条）。
- **intent**：`search.events`、`match.events`。
- **数据**：用线上真实赛事（见 PRD §16），首屏精选≤3 张。

## P0-2 赛程工作台 Schedule（合并原「首页状态面板」）

- **目标**：一眼回答"我现在的黑客松状态 + 下一步做什么"，不是日历堆砌。
- **核心模块**：活跃赛事状态面板（赛事名/距提交 X 天/进度%/7 阶段进度条/下一任务）→ 关键任务清单（≤3，勾选框）→ 提醒卡 → 推荐 AI 动作。
- **关键交互**：任务勾选；阶段点击看详情；提醒可关；状态面板点进赛事详情。
- **状态**：有活跃赛事 / 无赛程（引导发现 + AI 匹配）/ 截止临近（紧急红 `#FF4D2E` 倒计时）。
- **intent**：`schedule.status`、`event.remind`。

## P1-1 AI 匹配聊天 Chat

- **目标**：用自然语言对话帮用户匹配赛事，核心壁垒页。
- **核心模块**：对话流（用户气泡右/AI 回复左）→ 结构化追问 chips（城市/赛道/阶段）→ 结果以**内嵌赛事卡**呈现 → 底部输入栏（+上下文/scope/发送）。
- **关键交互**：流式输出 + 停止生成；点 chip 快速回答；点结果卡进详情；`src=ai` 落地时**延续微信 AI 上下文，不冷启动**。
- **状态**：首轮（3-4 个建议 prompt）/ 追问中 / 出结果 / 无合适赛事 / 单条失败可重试。
- **intent**：`match.events`、`project.match`。

## P1-2 匹配结果 Match Results

- **目标**：展示项目画像 + Top N 赛事匹配分。
- **核心模块**：项目画像卡（名称/技术栈 chips）→ Top 5 赛事卡（排名 #1-5 + 匹配分 + 进度条 + 推荐赛道）。
- **关键交互**：卡片点进详情；匹配分颜色阈值（≥85 成功绿 `#13B36B` / 60-84 主蓝 `#0C51ED` / <60 弱灰）。
- **状态**：loading / 有结果 / 无项目数据（引导同步或描述项目）。
- **intent**：`project.match`、`match.events`。

## P1-3 黑客松详情 Event Detail

- **目标**：给足信息让用户决策 + 行动；支持被 AI 深链直达（context-free）。
- **核心模块**：标题 → 关键事实（日期/倒计时/地点/形式/奖池 chips）→ 简介 → 赛道 → 赛程/适合人群预览 → AI 适配理由 → 底部双 CTA。
- **关键交互**（J-20~26）：**主 CTA「去官方报名」**（webview 开真实报名链接）+ 次 CTA「加入我的赛程」/ 设提醒 / 问 AI 这个适合我吗（带本赛事上下文进聊天）/ 生成参赛身份卡 / 创建提交清单；自带返回首页锚点（J-93）。
- **状态**：正常 / 报名已截止 / 未登录（行动时才登录）/ 半屏态（首屏给核心结论）。
- **intent**：`event.detail`、`event.remind`。

## P2-1 身份卡工坊 Identity Card

- **目标**：生成可复用的选手身份卡（匹配/组队/展示）。
- **核心模块**：身份卡/配置卡 切换 → 大卡片（角色/头像/技能/统计：项目·比赛·获奖）→ AI 判定角色 → 主技术栈 chips → 底部（保存图片/分享找队友）。
- **关键交互**：一键生成（预填微信资料+同步数据）；编辑；导出分享图；发布公开主页。
- **状态**：空（一键生成）/ 已生成 / 编辑中 / 未登录。
- **intent**：`identity.generate`。

## P2-2 Skills 同步 Sync

- **目标**：把桌面 CLI 扫描结果同步到小程序。
- **核心模块**：已同步结果卡 → 6 位配对码输入 → "如何获得配对码"步骤（装 Skill→进目录→运行 /ht-scan-project→复制码）。
- **关键交互**：输入码 → 同步；同步结果可应用到身份卡/项目。
- **状态**：输入 / 等待 / 成功（展示拉到的内容）/ 失败 / 过期。
- **intent**：`skills.sync`。

## P0-3 消息中心 Inbox（第 3 个 tab）

- **目标**：聚合提醒并把用户带回正确的行动页，不做泛泛通知流。
- **核心模块**：三分组（截止提醒 / 匹配结果 / 系统）→ 未读红点 → 一张焦点紧急提醒卡（如「AdventureX 报名截止仅剩 3 天」）→ 安静的历史消息行。
- **关键交互**（J-40~42）：截止提醒 → 详情或赛程；匹配结果 → 04（带 `matchId`，过期落「重新匹配」态）；系统消息 → 设置/说明。
- **状态**：正常 / 空态（CTA「去发现比赛」→ 发现 tab）/ 未登录（提示登录后接收提醒）。
- **intent**：`event.remind` 的结果承接。

## P0-4 我的 Profile（第 4 个 tab，聚合页）

- **目标**：个人资产中心，分组清晰不堆砌。
- **核心模块**（按优先级四分组）：① 首屏身份卡入口大卡（头像/名字/角色 + 「查看身份卡」）② 参赛资产（当前赛事 / 项目作品集 / 收藏·报名清单）③ 工具与同步（Agent·Skills 配置库 / Skills 同步）④ 设置（底部弱化）。
- **关键交互**（J-55/56）：身份卡大卡 → 06；各资产行 → 对应子页；收藏清单项 → 详情，报名清单项 → 详情或赛程。
- **状态**：已登录 / 未登录（游客态：示例卡 + 登录 CTA，不空屏）。
- **intent**：`identity.generate` 入口。
