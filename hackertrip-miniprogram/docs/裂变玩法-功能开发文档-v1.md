## 1. F1 — Haki 暗号（组队雷达 + 双向解锁）

### 1.1 需求 & 指标
- **目标**：让选手自发拉新，且新用户首次体验即获得 AI 个性化价值。
- **北极星**：暗号核销数（K 因子）、招募官解锁率、暗号带来的次日留存。
- **核心指标**：暗号生成率、暗号分享率、redeem 成功率、人均招募值。

### 1.2 角色 & 场景
- **邀请人 A**（已有用户）：想找队友/想解锁招募官卡 → 分享暗号。
- **被邀请人 B**（新/老用户）：收到暗号 → 丢给 Haki → 获得组队雷达 + 限定卡。

### 1.3 交互链路
```
A: 身份卡页「我的暗号」→ 复制邀请语 → 发群/朋友圈
B: 打开 Haki → 输入/粘贴 HT-XXXX（或点「🔑 我有好友暗号」弹框输入，或扫A的卡带 ?invite= 直达）
  → 前端正则识别 → 登录守卫(云端模式需登录) → api.redeemInvite(code)
    ├─ ok=false(NOT_FOUND/SELF/BAD_FORMAT) → Haki 气泡friendly报错，结束
    └─ ok=true → 组装 inviteContext{inviter, me} → aiChat(prepare, inviteContext)
        → Haki 流式输出「组队雷达」(识别+契合度+建议赛事+锐评)
        → toast：🎉 解锁「被邀请限定卡」
A: 招募值+1（满3解锁招募官卡）+ notifications 收到「有人用你暗号加入」
```

### 1.4 页面 & 组件
| 文件 | 改动 |
|---|---|
| `pages/identity/identity.{js,wxml,wxss}` | 「我的暗号」卡：`loadInvite()`/`copyInvite()`/`copyInviteCode()`/`buildInviteText()`；data.growth |
| `pages/chat/chat.js` | `matchInviteCode()`正则、`openInvitePrompt()`、`handleInvite()`、`localRadarText()`；`sendMessage`/`useQuickReply` 拦截暗号；`streamReply`/`fallbackReply` 透传 `opts.inviteContext`；onLoad 支持 `?invite=` |

### 1.5 数据模型
**集合 `invites`**
| 字段 | 类型 | 说明 |
|---|---|---|
| code | string | 暗号 `HT-XXXX`（去易混字符 base32，唯一索引） |
| ownerOpenid | string | 暗号所有者（索引） |
| ownerPublicId | string | 冗余公开 id |
| redeemCount | number | 被核销次数 |
| redeemedBy | string[] | 已核销的 openid（去重，防重复计数） |
| createdAt / lastRedeemAt | number | 时间戳 |

**集合 `users` 增量字段**：`invitedBy`(被谁邀请，只认首次)、`recruitScore`(招募值)、`unlockedCards`(服务端授予卡 key 数组)。

**storage `ht_growth`**：`{ inviteCode, recruitScore, redeemCount, invitedBy, unlockedCards[] }`（登录态隔离；登出清除）。

### 1.6 云函数接口
**`inviteCode`**（无入参，取 openid）
- 出参：`{ ok, code, recruitScore, redeemCount }` | `{ ok:false, message }`
- 规则：一个 openid 一个稳定暗号；不存在则生成唯一码（冲突重试 8 次，再兜底时间戳）。

**`redeemInvite`** `{ code }`
- 出参：`{ ok:true, firstTime, alreadyRedeemed, inviter:{name,role,city,bio,skills,recruitScore}, unlocked:[], ownerUnlocked:[] }`
- 失败码：`BAD_FORMAT` / `NOT_FOUND` / `SELF`
- 规则：归一化 code（容错大小写/缺连字符）；禁止用自己的暗号；同一 B 对同一 code 只首次计数；首次核销 → invite.redeemCount++ & redeemedBy addToSet & B.invitedBy(首次) & B.unlockedCards += invited_limited & A.recruitScore++ &（满 3）A.unlockedCards += recruiter & 写 A 的 notifications。

**`aiChat`（升级）**：`prepare`/`generate` 均接受 `event.inviteContext`。存在时在 system prompt 追加「组队雷达任务」段（识别邀请人→契合度%→从真实在办赛事推荐一场→锐评，180 字内，不追问）。
- `inviteContext = { inviter:{name,role,city,skills}, me:{role,city,skills} }`

### 1.7 降级 & 边界
- **未连云**：`getInviteCode` 用 publicId/openid 派生本地稳定码；`redeemInvite` 返回示例邀请人（Neo），`handleInvite` 走 `localRadarText` 兜底，开发者工具可完整演示。
- **云函数未部署/网络错**：`redeemInvite` 捕获异常后落到 mock 演示（不中断体验）；结构化校验失败(NOT_FOUND/SELF)则如实透出。
- **未登录**（云端模式）：`handleInvite` 先 `requireAuth`，带 `?invite=code` 回跳，登录后自动继续。
- **内容安全**：aiChat 对消息做 msgSecCheck（假 openid 会被拦，真实用户正常）。
- **防刷**：redeemedBy 去重；自邀拦截；招募值只随首次核销 +1。

### 1.8 埋点建议
`invite_code_view`(暗号曝光)、`invite_copy`(复制邀请语)、`invite_redeem_attempt/success/fail{reason}`、`radar_render`、`unlock{card}`、`recruiter_unlocked`。

### 1.9 验收标准
- [x] 身份卡页展示暗号 + 复制邀请语可用
- [x] Haki 识别 HT-XXXX（大小写/缺连字符容错）
- [x] 真实登录用户：redeem 成功 → Haki 输出组队雷达 → 解锁 toast → A 招募值+1
- [x] 自邀/错码/重复核销有正确反馈
- [x] 未连云可 mock 演示；编译零报错

## 2. F2 — 集卡卡册（稀有度成就卡）

### 2.1 需求 & 指标
- **目标**：把一次性身份卡升级为可收集体系，提升复访与炫耀传播。
- **指标**：卡册访问率、解锁卡数分布、稀有卡分享率。

### 2.2 卡面体系（`utils/unlocks.js`）
| key | 名称 | 稀有度 | 解锁条件 | 来源 |
|---|---|---|---|---|
| invited_limited | 被邀请限定 | SR | 通过好友暗号加入 | server |
| recruiter | 招募官 | SR | 招募值≥3 | server |
| first_blood | 首战纪念 | R | 报名第1场 | derive |
| streak_3 | 三连击 | SR | 报名满3场 | derive |
| hexagon_master | 六边形大师 | SSR | 技术栈跨4领域 | derive |
| synced | Skills接入 | R | 完成1次Skills同步 | derive |

稀有度：N/R/SR/SSR（`RARITY_META` 含色值）。

### 2.3 解锁判定（纯函数，mock 友好）
`deriveUnlocked(ctx)`：`serverUnlocked`(来自 ht_growth.unlockedCards) ∪ 由 `stats.hackathons`/`techCategoryCount`/`hasSync` 派生。`buildCardbook(ctx)` 产出带 `unlocked/rarityLabel/rarityColor` 的视图模型 + `unlockedCount/total`。
- ctx 来源：`api.getProfile().skills`(→`countTechCategories`)、`api.getUserStats().hackathons`、`api.getGrowth().unlockedCards`、`api.getScanResults().syncedAt`。

### 2.4 页面
- 新页 `pages/cardbook/{js,wxml,wxss,json}`（已注册 app.json）：角色 hero + 解锁进度条 + 2列稀有度网格(剪影/角标) + CTA「🔑 用好友暗号解锁限定卡」。`tapCard` 未解锁弹解锁指引并跳对应入口。
- `pages/profile/profile.js` tools 增「卡册 · 成就解锁」入口。
- `utils/roles.js` 导出 `countTechCategories`。

### 2.5 降级 & 验收
- 全程本地纯函数，无云依赖；未登录显示默认角色 + 全锁。
- [x] 卡册展示已解锁/未解锁 + 进度 + 解锁条件；点卡有指引；编译零报错。

### 2.6 迭代位

## 3. F3 — 赛事热度 + 主办方一键认领

### 3.1 需求 & 指标
- **目标**：把选手关注沉淀为赛事热度并可视化，反向吸引主办方认领/入驻。
- **指标**：详情页热度曝光、打call分享率、认领点击数、认领→入驻转化。

### 3.2 交互链路
- 选手：detail 顶部热度看板（🔥热度值 + 关注/报名/收藏 + 进度条）→「📣 为它打call」分享(文案带热度数) / 收藏报名贡献热度。
- 主办方：「这是你的赛事？认领」→ 弹窗(展示已有关注数) → `pages/organizer?claim=<id>` 承接。

### 3.3 数据 & 接口
**`getHackathonHeat`** `{ id }` → `{ ok, id, registrations, bookmarks, fans, heat }`
- 热度值 `heat = registrations*3 + bookmarks`；数据源 `registrations`(where id) + `bookmarks`(where hackathonId) 的 `.count()`。
- `api.getHackathonHeat(id)`：云端优先；**mock 降级**用 id hash 派生拟真值 + 叠加本人收藏/报名，开发者工具可演示。

### 3.4 页面
- `pages/detail/detail.{js,wxml,wxss}`：data.heat；`loadHeat(id)`(算 pct，软上限 300 封顶100)；`claimHackathon()`；`onShareAppMessage` 注入热度文案。

### 3.5 降级 & 验收
- 云失败/未连云走本地拟真值，不阻断详情。
- [x] 详情展示热度看板 + 打call分享 + 认领入口跳转；getHackathonHeat 线上实测返回 ok；编译零报错。

### 3.6 迭代位

## 5. 风险 & 对策
| 风险 | 对策 |
|---|---|
| 内容安全拦截组队雷达 | 雷达内容由 Haki 生成且经 msgSecCheck；prompt 约束正向；异常降级兜底文案 |
| 暗号被恶意刷量 | redeemedBy 去重 + 自邀拦截；后续可加频率限制/同设备风控 |
| 招募值写入并发 | 用 `_.inc` 原子自增，避免读改写竞态 |
| 云函数未部署导致 redeem 失败 | api 层捕获异常降级 mock，不中断；上线务必确认部署 |
| 集合不存在导致 add 报错 | invites/notifications 已建；notifications 写入额外 try/catch 不阻断主流程 |
