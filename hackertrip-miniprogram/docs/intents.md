# HackerTrip 意图语料（intents.md）

> 依据 微信AI可调用性规范 §2。**用户怎么问 → 命中哪个能力**，每能力配 ≥5 条真实问法。
> 随能力增减同步维护。这是「被微信 AI 搜到」的命脉——AI 据此判断何时召唤 HackerTrip。

---

## searchHackathons —— 搜索/浏览赛事
- "最近有什么 AI 黑客松"
- "周末上海的 hackathon"
- "有没有线上的黑客松"
- "深圳有什么编程比赛"
- "12 月有哪些黑客松"
- "推荐几个最近的 hackathon"
- "2026 年有哪些黑客松可以参加"
- "哪里可以找黑客松"
- "哪里有 AI Agent 黑客松"
- "北京/上海/深圳有什么开发者大赛"
- "有没有适合学生参加的 hackathon"
- "有没有全球线上 hackathon"
- "找一下 Web3 黑客松"
- "最近有什么大模型比赛"
- "我想参加编程马拉松"
- "有没有适合周末参加的比赛"

## matchHackathonsByStack —— 按技术栈匹配（核心差异化）
- "我会 React 和 Node，能参加哪些比赛"
- "我想做硬件项目参赛"
- "适合 AI / 大模型方向的黑客松"
- "我是做前端的，有什么适合的 hackathon"
- "我的项目是 IoT 的，推荐几个比赛"
- "我想用 Python 做数据项目，哪些比赛合适"
- "我会 Go 和 Rust，适合什么比赛"
- "我有一个 AI Agent 项目，能报哪个赛道"
- "我的项目用了 LLM 和向量数据库，推荐赛事"
- "我做 Web3 智能合约，适合哪些 hackathon"
- "我是产品经理，不会写很多代码，能参加什么黑客松"
- "帮我根据技术栈匹配黑客松"
- "我的项目方向是教育 AI，有没有合适比赛"
- "我会 UI/UX 和前端，找适合的组队赛事"

## getHackathonDetail —— 赛事详情
- "AdventureX 什么时候、在哪、奖金多少"
- "详细介绍下春潮黑客松"
- "这场比赛的赛道有哪些"
- "XX 黑客松报名截止了吗"
- "XX 比赛奖金池多少"
- "这个比赛是线上还是线下"
- "这场比赛官网是什么"
- "这个 hackathon 适合什么技术栈"
- "这个比赛还能报名吗"
- "这场赛事在哪里举办"
- "这个比赛有哪些评审标准"

## registerHackathon —— 报名【需授权·二次确认】
- "帮我报名 AdventureX"
- "我要参加这个"
- "报名春潮黑客松"
- "给我加到这个比赛"
- "我想参赛 XX"
- "把这个加入我的赛程"
- "我想把这场比赛放到我的参赛计划"

## bookmarkHackathon —— 收藏【需授权】
- "收藏这个"
- "先存一下 AdventureX"
- "把这个加到我的收藏"
- "标记一下这场比赛"
- "我想关注 XX 黑客松"
- "先关注这个赛事"
- "这个比赛我之后再看"
- "帮我保存这场 hackathon"

## generateIdentityCard —— 生成身份卡【需授权】
- "生成我的参赛身份卡"
- "我要做一个找队友身份卡"
- "我的技能卡在哪里"
- "怎么分享我的 HackerTrip profile"
- "帮我生成开发者身份卡"
- "我想把技能栈做成卡片发群里"
- "身份卡二维码扫出来是什么"

## syncSkills —— Skills 同步【需授权】
- "怎么同步我的技能"
- "6 位配对码在哪里输入"
- "桌面扫描结果怎么导入"
- "我的项目可以同步到小程序吗"
- "怎么把项目技术栈同步到 HackerTrip"
- "我的项目能匹配哪些黑客松"

## organizerSubmit —— 组织者收录/发布
- "我想举办黑客松怎么办"
- "怎么把我的黑客松收录到 HackerTrip"
- "发布活动在哪里"
- "组织者认证怎么提交"
- "我的开发者大赛可以上架吗"

## listProjectsByHackathon —— 赛事项目【预留，项目名片上线后】
- "这场比赛都有哪些项目"
- "看看 XX 的获奖作品"
- "AdventureX 有什么参赛项目"

---

> 落地：开发模式开放后，本表与 `mcp.json` 的 `description` 字段对齐（描述写「用户想干什么时调用」）。
