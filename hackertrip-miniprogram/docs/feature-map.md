## 一、四大 Tab 主页（底部导航栏，switchTab 切换）

| 页面 | 定位（一句话） | 核心功能 | 数据来源 | 状态 |
|---|---|---|---|---|
| **发现** `index` | 赛事浏览的总入口 | 搜索 / 筛选(全部·线上·线下·AI·Web3) / 卡片→详情 / 收藏 / 精选推荐 | 云函数 getHackathons（降级本地） | ✅ |
| **赛程** `schedule` | "我加入的"赛事进度管理 | 我的赛程列表 / 7段进度条 / 今日待办 / 空态引导 | 本地 registrations + 云函数补全 | ✅ |
| **消息** `inbox` | 通知聚合中心 | 截止提醒 / 匹配结果 / 系统通知 三分组 | 部分真实(赛事)+部分 mock(通知文案) | 🟡 通知系统未接 |
| **我的** `profile` | 个人资产与工具中心 | 参赛资产统计 / 身份卡入口 / 工具列表 / 设置入口 | 真实(收藏/报名/卡片计数) | ✅ |

## 二、赛事浏览流程页（navigateTo，左上返回）

| 页面 | 定位 | 核心功能 | 数据来源 | 状态 | AI intent |
|---|---|---|---|---|---|
| **详情** `detail` | 单个赛事的完整信息 | 赛事信息全字段 / 去官方报名(webview) / 加入赛程 / 问AI | 云函数 getHackathonDetail | ✅ | event.detail |
| **聊天** `chat` | Haki AI 对话助手 | 对话气泡 / 快捷回复 / 推荐赛事卡 | 赛事真实，**AI 回复是固定文案** | 🟡 AI 未接算法/模型 | match.events |
| **匹配结果** `match` | 按技术栈匹配赛事 | 项目卡 / Top赛事排名 / 命中理由 / 匹配度 | 真实(规则化计分算法) | ✅ | project.match |

## 三、身份与资产页（个人品牌体系）

| 页面 | 定位 | 核心功能 | 数据来源 | 状态 |
|---|---|---|---|---|
| **身份卡** `identity` | 生成参赛身份卡(裂变核心) | canvas 绘制角色卡 / 技术栈/打法/组队表单 / 保存相册 / 分享 | 统一用户档案 | ✅ |
| **身份编辑** `identity-edit` | 编辑统一档案 | 头像(chooseAvatar)/昵称/角色/城市/技能/GitHub | 统一用户档案(写) | ✅ |
| **作品集** `portfolio` | 项目作品展示 | 项目卡列表(标题/赛事/状态/技术栈) | **mock 数据** | 🟡 未接真实作品 |
| **Agent 技能库** `agent` | Haki 可调用技能配置 | 技能卡(启用/授权状态) | **纯展示** | 🟡 无交互 |
| **公开主页** `public-site` | 对外展示个人主页(GEO/分享) | 个人卡 / 资产 / 技能 / 项目 / 参赛历史 | 统一档案 + 真实赛事 | ✅ |

## 五、数据架构（统一数据层 utils/api.js）

```
页面 → api.js（唯一数据入口）→ 云函数 → 云数据库(实时)
                              ↘ 失败降级 → 本地 hackathons.js（真实兜底）
```

| 数据类型 | 云函数 | 本地存储 key | 状态 |
|---|---|---|---|
| 赛事列表/详情 | getHackathons / getHackathonDetail | data/hackathons.js | ✅ 10条真实(云=本地一致) |
| 收藏 | toggleBookmark | ht_bookmarks | ✅ |
| 报名/赛程 | addRegistration | ht_registrations | ✅ |
| 身份卡 | saveCard | ht_cards | ✅ |
| 用户档案 | getProfile/saveProfile | ht_profile | ✅ |
| 登录态 | login | ht_auth | ✅ |
| Skills 同步 | pairSync | ht_scan_results | ✅ |
