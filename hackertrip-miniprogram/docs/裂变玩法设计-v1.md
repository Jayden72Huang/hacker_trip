# HackerTrip 小程序裂变玩法设计 v1

> 目标：在现有「身份卡 + Haki + 黑客松聚合」基础上，设计**可落地、可执行**的裂变玩法，驱动「选手自发传播 → 拉新 → 平台热度 → 主办方主动入驻」的增长飞轮。
>
> 本文档先给战略框架与脑暴全集，再收敛到**落地前 3 条**（含 1 条 AI-native），最后给每条的交互链路、数据模型与落地清单。

---

## 0. 现状盘点（决定什么能落地）

| 现有资产 | 能力 | 对裂变的价值 |
|---|---|---|
| 身份卡 / 配置卡 | 10 角色 + secondary，Canvas 600×800，角色渐变卡面，内嵌小程序码 | 已是「可晒」原语，但**被动**，无解锁/收集/激励 |
| 小程序码 | `wxacode.getUnlimited` scene=`uid=<publicId>` → 落 `public-site` | 现成的**扫码进人主页**链路，scene 可携带 32 字符 |
| Haki（aiChat） | 混元大模型，已注入赛事+Agent 上下文，支持流式 | **AI-native 玩法的载体**：能识别、锐评、生成专属内容 |
| publicId / users 集合 | 每人唯一公开 id，公开主页 `getPublicProfile` | 邀请码/暗号、被访问统计的锚点 |
| registrations / bookmarks | 报名/收藏，云端双写 | 赛事**热度**的真实数据源 |
| inbox（消息页） | 纯计算列表，易注入 | 「被扫码/解锁/邀请成功」通知位 |
| organizer 流 | 申请 + 赛事草稿投稿 | B 端入驻的现成承接 |

**结论**：身份卡 + 小程序码 + Haki + publicId + 报名/收藏数据，已经凑齐裂变四要素（可晒物、传播介质、个性化反馈、热度数据），缺的是把它们串成**激励闭环**。

## 1. 脑暴全集（发散）

- **A. 身份卡稀有度 + 隐藏卡解锁/收集册**：角色卡加稀有度，新增成就限定卡面，卡册剪影驱动集卡。
- **B. 暗号/邀请码 + Haki 识别**：把好友暗号丢给 Haki，AI 识别此人并给惊喜锐评 + 双向解锁。
- **C. 组队雷达 / 契合度**：两张身份卡 → Haki 算组队化学反应、互补点。
- **D. 赛事选手墙 + 排行榜**：报名即进选手墙，城市/技术栈/招募榜。
- **E. 主办方热度看板 + 一键认领**：赛事热度页 + 战报海报，反向获客。
- **F. 邀请助力解锁**：拉 N 人解锁限定卡面/Haki 特权。
- **G. 线下扫码集卡**：黑客松现场互扫胸牌码，集满解锁现场限定徽章。

## 2. 收敛逻辑（为什么是这 3 条）

把 7 条按「**落地成本 × 增长杠杆 × 与用户明确诉求的契合度**」打分，合并同类项后收敛成三个互相咬合的飞轮齿轮：

```
F1 暗号邀请(AI拉新)  →  F2 集卡(留存+炫耀传播)  →  F3 热度看板(C端热度→B端获客)
      ↑                                                              │
      └───────────────────  更多赛事 → 更多选手  ←──────────────────┘
```

- **F1** 对应用户明确点名的「给 Haki 分享身份码 → 识别 → 惊喜反馈/解锁」，是 **AI-native 核心**，吃掉 B+C+F。
- **F2** 直接回应「身份卡功能延伸」，吃掉 A+G，是留存与炫耀传播引擎。
- **F3** 直接回应「让主办方看到热度并主动 post」，吃掉 D+E，是 B 端获客闭环。

三者共用同一套数据（publicId / 邀请 / 报名收藏），不是三个孤立功能，而是**一条增长链**。

## 3. F1 ⭐ AI-native 核心：「Haki 暗号」——身份码识别 + 双向解锁 + 组队雷达

### 一句话
每个人有一个**专属暗号**；把暗号发给 Haki，Haki 不止「查到这是谁」，而是用 AI 生成一段**针对你俩的组队锐评 + 惊喜反馈**，同时给邀请人记一笔「招募值」、给被邀请人解锁「被邀请限定卡」，形成双向激励的病毒裂变。

### 为什么有效（裂变心理学）
- **低门槛传播**：分享的不是链接而是一句「我的暗号 `HT-7K3D`，发给 Haki 试试」，对话气泡天然适合在群里发。
- **个性化惊喜**：AI 锐评是**不可预测的内容奖励**，每次都想看 Haki 怎么说 → 愿意晒结果。
- **双向获利**：A 得招募值/解锁，B 得限定卡 + 首次深度锐评 → 双方都有动机。
- **互惠循环**：B 想邀请别人也能拿到，于是去拿自己的暗号 → 指数扩散。

### 交互链路（端到端）
1. **生成暗号**：身份卡页/分享页新增「我的暗号」区块 → 调 `inviteCode`（云函数）由 publicId 派生 6 位短码（如 `HT-7K3D`），落 `invites` 集合，附一句可复制的邀请话术。
2. **传播**：A 把「暗号 + 话术」发到群/朋友（也可生成带暗号的卡片）。
3. **识别**：B 打开 Haki，置顶有「输入好友暗号」入口；B 把 `HT-7K3D` 发给 Haki。
4. **AI 反馈**：前端识别到暗号 → 先调 `redeemInvite`（云函数）拿到 A 的角色/技术栈/城市/招募意向 + 绑定 referral → 把这些作为 context 注入 Haki，让 Haki 生成：
   - 「你被 **{A 角色}** 邀请」开场
   - **组队雷达**：你俩契合度 X%、互补点、建议一起打哪个在办赛事（用真实赛事数据）
   - 一段有梗的锐评（Haki 人设）
5. **解锁结算**：
   - B：解锁「被邀请限定卡面（青金特效）」+ 记录 invitedBy=A。
   - A：招募值 +1；达阈值（3 / 10）解锁「招募官」称号卡面；inbox 收到「{B} 通过你的暗号加入了 HackerTrip 🎉」。
6. **回环**：B 在结果页看到「生成我的暗号，邀请朋友也能解锁」→ 回到第 1 步。

### AI-native 体现
不是查表回话，而是：暗号→结构化 context→**Haki 用自然语言现场生成专属组队分析与锐评**，结合真实在办赛事给「一起去打 XX」的行动建议。内容千人千面、不可预测，这是只有 LLM 能给的体验。

### 数据模型
- 新集合 `invites`：`{ code, ownerOpenid, ownerPublicId, createdAt, redeemCount, redeemedBy:[openid] }`
- `users` 增量字段：`invitedBy`（被谁邀请）、`recruitScore`（招募值）、`unlockedCards:[]`（已解锁卡面 key）
- 防刷：同一 openid 对同一 code 只计一次；不能用自己的暗号。

### 落地清单
- 云函数：`inviteCode`（建/取暗号）、`redeemInvite`（核销+绑定+返回 inviter 画像与契合上下文）。
- aiChat：`prepare` 支持 `inviteContext`，把组队雷达 context 注入 system prompt。
- 前端：identity/share 加「我的暗号」区块；chat 加暗号识别与置顶入口；结果解锁动效复用 Canvas。
- inbox：新增「邀请成功 / 被解锁」通知项。

## 4. F2：身份卡稀有度 + 隐藏卡收集册（Collection）

### 一句话
把「一张身份卡」升级成「一套要集齐的卡」：10 角色卡加稀有度，新增**成就限定卡面**（被邀请卡、城市限定、赛事纪念卡、连胜卡、招募官卡…），卡册里未解锁显示剪影 + 解锁条件，制造收集与炫耀动机。

### 为什么有效
- **收集癖 + 进度可视**：剪影 + 「已解锁 6/12」制造完成欲。
- **稀有度炫耀**：晒出稀有卡 = 晒成就，别人想要同款 → 进来集卡。
- **复用现有数据零成本触发**：解锁条件直接挂在报名数、邀请数（来自 F1）、被扫码数、Skills 同步等已有事件上。

### 卡面体系（初版）
| 卡面 key | 名称 | 解锁条件 | 稀有度 |
|---|---|---|---|
| role_* | 10 个角色卡 | 生成身份卡即得主角色卡 | N/R |
| invited_limited | 被邀请限定 | 通过暗号被邀请加入（F1） | SR |
| recruiter | 招募官 | 招募值 ≥ 3 | SR |
| city_* | 城市限定 | 在某城市报名线下赛 | R |
| first_blood | 首战纪念 | 报名第 1 场赛事 | R |
| streak_3 | 三连击 | 报名 3 场 | SR |
| hexagon_master | 六边形大师 | 技术栈跨 ≥ 4 领域 | SSR |

### 交互链路
- profile/「我的」新增「卡册」入口 → 卡册页网格：已解锁高亮、未解锁剪影 + 「再报名 1 场解锁」式提示。
- 点已解锁卡 → 大图预览 + 保存/分享（复用 identity 的 Canvas 导出）。
- 解锁瞬间：弹「叮！解锁 {卡名}」动效 + inbox 通知 + 引导分享。

### 数据模型
- `users.unlockedCards: [key]`；解锁逻辑可在前端由现有资产**纯函数派生**（`deriveUnlocks(profile, stats, invites)`），云端只持久化结果，保证 mock-first 可跑。
- card-canvas 增加 `face` 参数支持限定卡面（额外角标/特效/底纹），不破坏现有 identity/config。

### 落地清单
- utils：新增 `unlocks.js`（卡面定义 + 解锁判定纯函数）。
- card-canvas：扩展 `drawCard` 支持 `face`/`rarity` 角标与特效。
- 新页：`pages/cardbook/cardbook`。
- profile：加「卡册」入口与已解锁数角标。

## 5. F3：赛事热度看板 + 主办方一键认领（B 端获客飞轮）

### 一句话
把选手端真实的关注/收藏/报名沉淀成每个赛事的**热度**，对选手做「为赛事打 call」的传播，对主办方做「这是你的赛事？认领并补全信息」的极低门槛入驻引导，并生成**热度战报海报**让主办方愿意主动晒、主动 post。

### 为什么有效
- **数据即信任**：主办方最在意「有没有人来」。把「147 人关注 / 38 城选手画像」摆给他看，比任何 BD 话术都有效。
- **C→B 反向获客**：选手热度可视化 → 主办方看到 → 主动认领/发布 → 平台供给增加。
- **海报自传播**：主办方晒「我的赛事在 HackerTrip 热度 TOP3」= 帮我们做品牌曝光。

### 交互链路
- **选手侧**：detail 页加「热度条」（关注/报名意向/选手技术栈分布）；「为这个赛事打 call」按钮 → 报名/收藏即 +热度 → 生成赛事热度海报可分享。
- **主办方侧**：detail 页底「这是你举办的赛事？认领」→ 进 organizer 认领流（复用现有申请）→ 认领后可看完整热度看板（浏览/收藏/报名画像）+ 补全赛事信息（复用 hackathon-create）。
- **战报海报**：赛事热度页一键生成海报（赛事名 + 热度数 + 选手画像 + 小程序码），主办方/选手都可晒。

### 数据模型
- 热度由 `bookmarks` + `registrations` 按 hackathonId 聚合（云函数 `getHackathonHeat`，mock 时本地统计）。
- 可选 `hackathon_views` 计数（轻量）。

### 落地清单
- 云函数：`getHackathonHeat`（按赛事聚合关注/报名数 + 技术栈/城市分布）。
- detail：热度条 UI + 打 call + 认领 CTA。
- 海报：复用 Canvas 出赛事热度海报。
- organizer：认领入口承接（复用 submitOrganizerApplication）。

## 6.5 调研印证（Bonjour + 小程序 playbook + B 端）

并行调研（Bonjour 官网/创始团队播客、群勾搭分销拆解、网易「请拆穿」刷屏案例、Devpost/DoraHacks B 端价值主张）**强力印证**本设计三齿轮，并补充了优化点：

- **印证 F1**：群勾搭验证「即时反馈 + 实时排行榜」是裂变最大成瘾点（结算从 7 天改为秒到 + 排行榜，单人拉新从 25 升到 50-200）。→ 已把 **招募值即时可见**（暗号卡/卡册都显示「招募 N 人」）；**下一步加「本周拉新黑客榜 / 本场赛事邀请王」排行榜**。
- **印证 F1 的 AI 形态**：网易「请拆穿」三层结构（自我表达 → **好友判定** → 排名 PK）是最易引爆朋友圈的单点。我们的 **Haki 组队雷达就是 AI 版「好友判定」**（好友把暗号丢给 Haki，Haki 现场判定你俩契合度），比静态测试卡更有不可预测的惊喜感。
- **印证 F2**：解锁式多层任务（转发→邀请→进阶权益）+ 稀有度收集是被反复验证的留存裂变。
- **印证 F3**：Bonjour/Devpost 的 B 端本质卖点都是「我这有开发者流量和报名转化」；**选手端热度是主办方信任的唯一硬通货**。→ F3 的「热度可视化 + 选手 UGC 带赛事标签 + 一键认领」完全踩中，**下一步加平台级大盘**：「本月 X 名开发者在 HackerTrip 找黑客松」制造主办方 FOMO。
- **Bonjour 专属可抄点**：① profile 即作品（身份卡颜值/信息密度是裂变地基）；② 线下「碰一碰/扫码交换开发者卡」沉淀「我在 XX 黑客松认识的人」——这是相对线上平台的差异化杀招，与黑客松现场天然契合，列入路线图。
