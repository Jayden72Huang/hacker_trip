# HackerTrip 小程序 · 上架指南

> 目标：从开发完成到通过微信审核上线。预计 1–3 个工作日（含审核）。

---

## 0. 前置条件

- [ ] 已注册**正式小程序**（个人主体即可），拿到 AppID
- [ ] 已开通云开发并部署全部云函数（见 README）
- [ ] `project.config.json` 的 appid 已改为正式 AppID
- [ ] 已在真机预览中跑通核心流程

---

## 1. 基本信息（mp.weixin.qq.com → 设置）

| 项 | 建议填写 |
|----|---------|
| **名称** | `HackerTrip` 或 `HackerTrip 黑客松`（需查重，重名时用后者） |
| **简介** | 见 `docs/商店文案.md` |
| **头像** | 用品牌紫色 Logo（512×512，见下方图标说明） |
| **服务类目** | 主：**工具 → 效率**；备选：**教育 → 在线教育**（黑客松信息/学习）。个人主体可选类目有限，工具/效率最稳 |

> ⚠️ 个人主体小程序**不支持**部分类目（如资讯、社区）。本程序定位「工具/效率」可过。

---

## 2. 用户隐私保护指引（必填，否则无法发布）

小程序后台 → 设置 → 服务内容声明 → **用户隐私保护指引**，按 `docs/隐私协议.md` 填写。
本程序涉及的隐私接口：
- `wx.saveImageToPhotosAlbum`（保存身份卡到相册）→ 声明「相册（仅写入）」
- 云开发 openid（用于跨设备同步卡片/收藏）→ 声明「设备标识/用户身份」

代码中已对相册权限做了 `wx.openSetting` 引导（`pages/card/card.js`）。

---

## 3. 合法域名 / 服务器配置

- 纯云开发**无需**配置 request 合法域名（云函数调用走内部通道）
- 若后续接官网 `hackertrip.space` 的 HTTP 接口：
  开发管理 → 开发设置 → 服务器域名 → request 合法域名加 `https://hackertrip.space`
  （该域名需已完成 **ICP 备案**）

---

## 4. 提交审核前自检

- [ ] 开发者工具「详情 → 本地设置」勾选 ES6 转 ES5、上传代码时样式自动补全
- [ ] 体验版真机测试：发现/详情/卡片生成/保存图片/同步/我的 全部正常
- [ ] 分享卡片到微信会话，落地页 `pages/share` 正常显示
- [ ] 无 `console.error`，无白屏
- [ ] 「上传」代码 → 后台「版本管理 → 开发版本」→ 提交审核
- [ ] 填写**审核备注**：说明「同步」配对码功能用于拉取用户自己电脑端的扫描结果，无第三方数据

## 5. 审核常见驳回点 & 规避

| 驳回点 | 规避 |
|--------|------|
| 功能不完整/页面空 | 已内置 15 条黑客松数据，保证非空；审核账号也能看到内容 |
| 诱导分享 | 分享文案为「看看你是什么角色」，非「分享得奖励」，合规 |
| 类目与内容不符 | 选「工具/效率」，主打黑客松检索 + 身份卡工具 |
| 隐私指引缺失 | 见第 2 步，必须配置 |
