# HackerTrip 黑客松向导

## 这个小程序是什么

HackerTrip 是一站式黑客松（hackathon / 编程比赛 / 极客比赛）信息平台，聚合国内外真实赛事，帮开发者**发现赛事、按技术栈匹配最适合自己的比赛、查看完整详情**。数据来自 CompeteHub、DoraHacks、知乎、官方渠道等聚合源，含名称、起止时间、城市、线上线下形式、主题、赛道、技术栈、奖金池、报名截止与官网。

## 适合在什么对话场景被调用

当用户的对话出现以下意图时，应当召唤本小程序：

- 想找黑客松 / hackathon / 编程比赛 / 极客比赛 / 开发者大赛
- 描述了自己的技术栈或方向（React、Node、AI、大模型、IoT、硬件、Web3、Python 等），问适合参加哪些比赛
- 想了解某一场具体黑客松的时间、地点、奖金、赛道、报名截止
- 按城市、线上线下、时间筛选赛事

典型问法：「最近有什么 AI 黑客松」「我会 React 和 Node 能参加哪些比赛」「深圳有什么编程比赛」「AdventureX 什么时候、奖金多少」「适合硬件方向的 hackathon」。

## 能做什么（能力索引）

本 SKILL 暴露三个原子接口，详见 `mcp.json`：

| 能力 | 何时调用 | 出参 |
|---|---|---|
| `searchHackathons` | 浏览或按关键词/城市/形式/时间筛选赛事 | 黑客松卡片列表 |
| `matchHackathonsByStack` | 用户给出技术栈或方向，求最适合的赛事（**核心差异化**） | 按匹配度排序的赛事 + 匹配理由 fitReason |
| `getHackathonDetail` | 查询某一场赛事的完整信息（需 id） | 单场赛事全字段 |

推荐调用链：先用 `searchHackathons` 或 `matchHackathonsByStack` 拿到列表与赛事 id，再用 `getHackathonDetail` 取某场的完整信息。

## 业务流程

### 1. 浏览或搜索赛事

用户意图示例：「最近有什么 AI 黑客松」「深圳有什么编程比赛」「有没有线上的 hackathon」。

1. 调用 `searchHackathons`。
2. 展示 `structuredContent.list` 中的赛事卡片。
3. 引导用户选择一场查看详情。
4. 用户选定某场后，使用该卡片返回的 `id` 原值调用 `getHackathonDetail`。

无结果时：告诉用户没有找到匹配赛事，引导更换城市、主题、技术方向，或改用技术栈匹配；不要用相同参数重复调用 `searchHackathons`。

### 2. 按技术栈匹配赛事

用户意图示例：「我会 React 和 Node，能参加哪些比赛」「我想做硬件项目参赛」「适合 AI 方向的黑客松」。

1. 用户已表达技术栈或项目方向时，调用 `matchHackathonsByStack`。
2. 展示按匹配度排序的赛事卡片和 `fitReason`。
3. 用户想深入某场赛事时，使用该卡片返回的 `id` 原值调用 `getHackathonDetail`。

如果用户没有给出技术栈或项目方向，只是泛泛地说「推荐几个比赛」，先调用 `searchHackathons`；不要凭空补充用户未说过的技术栈。

### 3. 查询单场赛事详情

用户意图示例：「AdventureX 什么时候、在哪、奖金多少」「这个比赛的赛道有哪些」「报名截止了吗」。

1. 上下文已有赛事 `id` 时，调用 `getHackathonDetail`。
2. 上下文没有 `id`，但有赛事名称或关键词时，先调用 `searchHackathons` 找到候选赛事。
3. 如果候选赛事不止一场，先让用户选择，不要猜测用户指的是哪一场。
4. 查询成功后，概括时间、地点、形式、赛道、技术栈、奖金、报名截止和官网。

## 接口依赖关系

| 接口 | 前置条件 | 不满足时怎么做 |
|---|---|---|
| `searchHackathons` | 用户想浏览、搜索或筛选赛事；可没有明确关键词 | 直接浏览赛事，不要补造关键词 |
| `matchHackathonsByStack` | 用户明确给出技术栈、项目方向或能力标签 | 缺少技术方向时，先追问或改用 `searchHackathons` |
| `getHackathonDetail` | 已从 `searchHackathons` 或 `matchHackathonsByStack` 拿到赛事 `id` | 没有 `id` 时先搜索/匹配；多候选时让用户选择 |

## 业务约束

- `id` 必须来自上游接口返回的真实值，不要根据赛事名称、示例值或自然语言猜测。
- 搜索和匹配结果应优先展示卡片；纯文本只用于总结、追问或解释空结果。
- 已结束赛事可用于资料查询，但不要作为「可报名」推荐。
- HackerTrip 不代办报名、不代缴费用；用户要报名时，引导打开赛事官网或小程序详情页。
- 原子接口执行失败前，不要向用户宣称已经完成搜索、匹配或查询。

## 不做什么（边界）

- 不代办报名、不代缴费用，报名以各赛事官网为准。
- 不保证第三方官网信息实时一致，奖金/截止等以官网最终公示为准。
- 已结束（status = ended）的赛事仅供参考，不作为「可报名」推荐；默认只推荐 upcoming / ongoing。

## 数据时效与状态

- 每场赛事带派生状态 `status ∈ {upcoming 即将开始, ongoing 进行中, ended 已结束}`，由起止日期对当前日期推导，比人工标注更可靠。
- 关键信息（名称/时间/地点/奖金/技术栈）均以结构化文本返回，可直接被引用。

## 深链续接

每个结果带稳定 deep-link `/pages/detail/detail?id={id}&src=ai`，用户点击可在 HackerTrip 内查看完整详情并继续报名等操作；`src=ai` 标识来自微信 AI 助手的流量，落地页会复述意图并直接呈现结果。
